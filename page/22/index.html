JDBC的配置解决中文乱码

发表于 2015-04-03   |   字数统计: 98(字)   |   阅读时长: 1(分) | 分类于 java |

在JDBC的配置数据库地址中添加上了两个参数参数useUnicode=true&character.Encoding=UTF-8。
添加上这两个参数是为JDBC在默认情况下会采用操作系统的默认编码和数据库通信，如果数据库采用的编码不是系统默认的编码，那就要进行显式指定，否则会发生中文乱码。
